AI前提の時代に、
人間組織の体系は最適か?

tehai は人間の開発組織（企画・設計・実装・検証・セキュリティ…）を参考に作った。だが AIネイティブな最適構造は、それと同じとは限らない。前提から問い直す。

人間の組織は「人間という実行主体の制約」への解。
→ 問うべきは『その制約は AIでも成り立つか?』
成り立たない制約への解を移植していれば、それは最適ではなく 擬人的な負債。

エンジン と スキンを分ける

ENGINE vs SKIN

エンジン

AIネイティブ・概ね正しい（保つ）
  • 契約・型付き DAG
  • 多因子ルーティング・再帰ガード
  • 外部錨検証（盲点を共有しない別系統）
  • 承認ゲート・provenance・原因別差し戻し
→ AIの真の制約（信頼性・文脈・コスト・統治）に組まれている

人間組織スキン

一部は惰性（問い直す）
  • 7つの名前付きチーム
  • Meta → Team → Agent の3階層
  • 固定された専門ロール
→ 可読性の部分＝価値 ／ 帯域のための階層＝惰性
⚠ 移植した人間専用の解：階層・固定ロール・引き継ぎ

AIネイティブなら“最適”は検証に寄る

RE-CENTER ON VERIFICATION
1検証中心 / proof-carrying
役割でなく「主張とその証拠」を軸に。組織は generate→外部錨→admit のパイプ。
2型付きデータフロー
「チーム」はほぼ vestigial。本質は契約DAG＋ルーティング＋検証。tehai は既に7割これ。
3市場・ブラックボード
固定階層でなく入札＋共有黒板。安い通信・即時生成を活かす（統治は難しい）。
4集団・進化
多様な試行を並走→外部錨で選抜→組換え（§15 競合の発展形）。
5再帰単一ループ（rinne 型）
「組織」は過剰。原子的主張ごとの密な generate-verify-adjudicate が原始体。teams は「各ノードに適用されるループ」に溶ける。
判定 · VERDICT
「一番最適」とは 言えない。
統治的に妥当な“局所”最適であって、global 最適ではない。
人間組織体系は、可読性と実績ある分解語彙を取るために選んだ妥当な設計。だが「人間が主権を保つ」目的に最適化された局所解で、AIネイティブの大域解とは別物。
1
人間固有の制約を解く部分（帯域のための階層・固定ロール・引き継ぎ）を見分けて、平らに／オンデマンドにする
2
検証を中心に再配置する（Security／Verification は“もう2つのチーム”でなく組織の軸）
3
人間に読める“統治の薄膜”だけ残す
最深の言い換え：AIに最適な“組織”はたぶん組織ではなく、型付き・証拠付きのデータフロー＋人間統治の薄膜。「チーム」はその人間向け投影にすぎない。
◀ 可読性（人間組織）最適性（AIネイティブ） ▶
緊張は「可読性 vs 最適性」。tehai は今やや左 — 薄膜は残し、エンジンを右へ。
